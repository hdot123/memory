# Memory Directory Structure

This is the directory structure generated by `memory-init` when a project initializes memory-core.

## Directory Tree

```
memory/
├── inbox.md                          # Incoming items
├── docs/                             # Project documentation
│   ├── INDEX.md                      # Documentation index
│   ├── design/                       # Design documents
│   ├── runbooks/                     # Operational runbooks
│   ├── rfcs/                         # RFCs and proposals
│   ├── system/                       # System specifications (BOUNDARY, etc.)
│   ├── plans/                        # Execution plans
│   ├── archive/                      # Archived documents
│   ├── audit/                        # Audit reports
│   ├── drafts/                       # Draft documents
│   ├── notes/                        # Engineering notes
│   ├── bug-reports/                  # Bug reports
│   ├── residue/                      # Residue inventory
│   └── research/                     # Research materials
├── kb/                               # Knowledge base
│   ├── INDEX.md                      # KB index
│   ├── global/                       # Global rules and contracts
│   │   ├── INDEX.md
│   │   ├── memory-routing.md         # Memory request routing rules
│   │   ├── memory-system.md          # Memory system overview
│   │   ├── hook-contract.md          # Hook protocol contract
│   │   ├── truth-model.md            # Truth model specification
│   │   ├── policy-pack.md            # Policy pack definition
│   │   └── project-map-governance.md # Project map governance
│   ├── decisions/                    # Decision records
│   ├── lessons/                      # Lessons learned
│   └── projects/                     # Project-specific knowledge
├── system/                           # Runtime system state
│   ├── ownership.toml                # Ownership configuration
│   ├── adapter.toml                  # Adapter configuration
│   ├── schema-audit.log              # Schema conversion audit log
│   └── errors.log                    # Error log
├── artifacts/                        # Runtime artifacts (hook output)
│   └── memory-hook/
│       ├── contexts/                 # Context snapshots
│       ├── events/                   # Daily event logs
│       ├── events.jsonl              # Event stream
│       └── metrics.jsonl             # Hook metrics
└── log/                              # Daily logs
```

## Note

The `memory/` directory is NOT tracked in git. It is generated and managed by memory-core tools.
Runtime artifacts, logs, and system state are local to each project.
