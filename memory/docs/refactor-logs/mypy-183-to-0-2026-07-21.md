## Milestone 1: Easy（56 errors，5 文件）

**PR**: #176
**修复**: 56 errors → 0

### 关键修复

#### 1. denylist.py denied() 方法重命名（19 errors）

**问题**：classmethod `denied()` 与 DenylistResult.denied 字段同名，mypy 无法消歧。

**解决**：重命名 `denied()` → `make_denied()`

```python
# 修复前
class Denylist:
    @classmethod
    def denied(cls, ...) -> DenylistResult:
        return DenylistResult(denied=True, ...)

# 修复后
class Denylist:
    @classmethod
    def make_denied(cls, ...) -> DenylistResult:
        return DenylistResult(denied=True, ...)
```

**影响**：3 处调用点更新（denylist.py 内部 + 2 个测试文件）

#### 2. consistency_check.py type-arg 修复

**问题**：泛型类型缺少类型参数

```python
# 修复前
def check() -> dict:  # mypy: Missing type parameters for "dict"
    ...

# 修复后
def check() -> dict[str, Any]:
    ...
```

#### 3. 小文件批量修复

- `_file_utils.py`: 4 errors（type-arg + annotation）
- `cmux_hook_state.py`: 6 errors（TypedDict + None check）
- `memory_hook_integrity_manifest.py`: 7 errors（cast + type narrowing）

## Milestone 2: Gateway（84 errors，1 文件）

**PR**: #177, #178
**修复**: 84 errors → 0

### 关键修复

#### 1. globals().update 动态注入问题（47 errors）

**问题**：`globals().update(profile)` 运行时注入 ~45 个常量名，mypy 报 name-defined errors。

**解决**：TYPE_CHECKING 块声明常量类型

```python
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    HOOK_VERSION: str
    PROJECT_ROOT: Path
    MEMORY_ROOT: Path
    # ... 45 个常量类型声明
```

**关键权衡**：
- ✅ TYPE_CHECKING 块：零运行时改动，+58/-3 行（PR #177）
- ❌ 重构为 config store：破坏性重构，影响 100+ 调用点
- ❌ `# type: ignore`：47 个 ignore 散布，维护成本高

#### 2. TypedDict 和 cast 修复（37 errors）

**问题**：profile 字典结构不匹配，类型转换需要显式 cast。

**解决**：
- 定义 ProfileDict TypedDict
- 使用 cast() 显式转换

```python
from typing import cast, TypedDict

class ProfileDict(TypedDict):
    name: str
    version: str
    # ...

def load_profile() -> ProfileDict:
    raw = _load_raw_profile()
    return cast(ProfileDict, raw)
```

**M2 门禁**: pytest 3111 passed / mypy 66 errors / vulture 86 / radon D+ 0

## 技术亮点

### 1. TYPE_CHECKING 模式标准化

为后续类似场景（动态注入、延迟加载）建立标准解决方案：

```python
if TYPE_CHECKING:
    # 声明动态注入的常量类型
    HOOK_VERSION: str
    PROJECT_ROOT: Path
```

### 2. vulture whitelist 集中管理

避免 `# noqa` 散布，便于审计：

```python
# vulture_whitelist.py
WHITELIST = ["CONSTANT_1", "CONSTANT_2", ...]
```

### 3. 渐进式 milestone 策略

Easy→Gateway→Scattered 降低合并冲突风险，每个 milestone 独立可回滚。

### 4. 零运行时改动

所有修复都不改变运行时行为，只增强静态类型信息。
